POC on Quarkus-Kafka with Postgres

Sabbyasachi Chakraborty

Question: A Quarkus application has to be created which will integrate with Postgres Database and Kafka Broker so that an Entity can be added in parts(Partial JSONs) getting identified whether being added or updated along with reading, updating or deleting the entity.
Steps :-

  • Created a Quarkus application based on the devon architecture (dataaccess, logic and service layered with api and implementations) which will receive the entity in parts identified by its ID (as JSONs through Postman) and identify and add it to database.
  • Configured the Postgres database along with the entity Repository(as JPARepository).
  • Integrated Kafka to produce the entity received and consume it to implement the logic.
  • Implemented the REST endpoints and all the possible test cases to check the intended working of the application.
Scope of improvement- More complex mappings need to be implemented with related entities and a fully functional frontend needs to be developed with Angular which can accept the entities and forward it to backend.
